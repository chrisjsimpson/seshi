About

What is Seshi?

Seshi is quite a sophisticated web application, but it is nothing new. It's simply how its architected which makes it different. Seshi doesn't need seshi.io in order to function, that is, the web server on which it runs - is not required in order for people to connect to each other. For example, you can save the webpage locally and run it from there. Signalling is still needed of course (how do users find each other?) and for this, Seshi provides signalling nodes. Seshi generates a key for you which can be exchanged outside of the application (a so-called out of band communication) to allow two peers to connect. Front end designer Anthony Ho is making Seshi look more awesome every day and without him Seshi may well have stayed just a geeky experiment.

The codebase is split into two parts: Seshi Skin & Seshi Signal, and we're working on making it easier for developers to build their own interfaces whilst still plugging into Seshi's signalling network & storage framework. If your interested in this, feel free to drop us a message or tweet!

Who are we?

Seshi Ltd is based in Newcastle, United Kingdom. The founder, Christopher Simpson put together Seshi whilst at University. The motivation behind Seshi was to create a truly decentralised file sharing tool which would put people back in control of their own content, and be able to exchange their files in a peer to peer fashion.

"Services like Seshi are possible thanks to an open, unencumbered web. The strides made with WebRTC are great example of this. It's no longer a viable business model to create incompatible experiences which require plugins or restrictive licences. Try and create a great experience, share the knowledge."
